# memoryd OpenClaw plugin

OpenClaw 一等公民 native plugin —— 把 OpenClaw 接入本地 memoryd 后端。

**v0.3 形态**：3 个原生工具 + 2 个 lifecycle hook，外加一条 lifecycle 事件桥接兜底。

| 能力 | 形式 |
|---|---|
| `memory_search(query, top_k?, scope?)` | native tool，调 memoryd hybrid search |
| `memory_get(memory_id)` | native tool，取一条 memory 的完整 markdown + frontmatter |
| `memory_transcript(memory_id)` | native tool，取 session 的原始 transcript |
| `before_agent_start` hook | 向 system message 注入 identity 节选 + top 实体 + 最近 5 条 long-term |
| `agent_end` hook | 异步抽摘要 → `memoryd capture --source=openclaw` |
| lifecycle 事件桥接（兜底） | 旧 SDK 没有 agent_end 时用 `register.js` 那条订阅 |

## 设计要点

- **零新依赖**：纯 Node ≥ 18 stdlib + `node:test`，没有 `npm install` 步骤。
- **memoryd 调用抽象** 在 `src/memoryd_client.js`：默认 spawn 本地 `memoryd` CLI；如果用户开了 Web Dashboard，可以切到 `transport: "http"` 调本地 `memoryd-server`。
- **fire-and-forget**：所有 capture/recall 失败都只写日志，不阻塞 agent；所以即便 memoryd 进程暂时挂了，OpenClaw 也能照常用。
- **不接管原生记忆**：plugin manifest 故意不设 `kind: "memory"`，OpenClaw 自带的 memory-core 仍然工作，我们做的是"额外的本地记忆层"。

## 安装

```bash
cd /Users/abble/memory-system/plugins/openclaw
openclaw plugins install --force .
```

授权（hook 需要看 conversation，工具不需要）：

```bash
openclaw config set plugins.entries.memoryd-openclaw.hooks.allowConversationAccess true
openclaw config set plugins.entries.memoryd-openclaw.hooks.allowPromptInjection true
```

可选配置（`~/.openclaw/config.toml`）：

```toml
[plugins.entries.memoryd-openclaw.config]
transport    = "cli"        # 或 "http"
binPath      = "/usr/local/bin/memoryd"  # 覆盖 MEMORYD_BIN
port         = 8765
autoCapture  = true         # 关掉就停止 agent_end 自动写
autoRecall   = true         # 关掉就停止 before_agent_start 注入
```

## 工具调用示例（agent 内）

```javascript
// 1) 找最近的相关记忆
const hits = JSON.parse(
  (await tools.memory_search({ query: "abble 切 Solid 决策", top_k: 5 }))
    .content[0].text
);
// hits: [{ memory_id, content_preview, scope, created_at, dura_score, source }]

// 2) 取详情
const md = (await tools.memory_get({ memory_id: hits[0].memory_id })).content[0].text;

// 3) 必要时回看 transcript
const trans = (await tools.memory_transcript({ memory_id: hits[0].memory_id })).content[0].text;
```

## hook 行为

### `before_agent_start`

异步并发拉三类数据后拼一段 markdown 注入 system message 末尾：

```
## 与 abble 的最近上下文
<!-- generated by memoryd openclaw plugin @ 2026-05-19T... -->

### Identity 节选
abble，36 岁，前端独立开发者……（≤300 字）

### 最近 30 天 top 实体
- wolin-admin (score=0.92) —— 切 Solid 中
- zhuzhen-team (score=0.71)
- ...

### 最近 5 条 long-term 记忆
- 2026-05-18 切 Solid 决策 [2026-05-18T...]
- ...
```

三类数据都拿不到时直接返回空对象，不污染 prompt。

### `agent_end`

1. 从 `event.messages` 抽最后 N 轮 user+assistant 对话
2. 喂注入式 LLM 摘要器（默认 naive 截断；上层可以替换成 `memoryd llm summarize` 调用）
3. 调 `memoryd capture --source=openclaw` 写 markdown

整个流程 fire-and-forget，**不 await 摘要+capture 完成**，避免拖慢 OpenClaw 主循环。

## 跑测试

```bash
cd /Users/abble/memory-system/plugins/openclaw

```

测试覆盖：

- `tests/payload.test.mjs` —— `buildPayload` / `materializeTranscript`（已有）
- `tests/register.test.mjs` —— lifecycle 兜底订阅（已有）
- `tests/tools.test.mjs` —— 3 工具的注册、参数 schema、execute 返回格式、错误透传
- `tests/hooks.test.mjs` —— 2 hook 的输入解析、注入格式、异步 capture 是否触发

所有 spawn / HTTP 调用都被注入 mock，测试不会真的去拉 memoryd 进程。

## 故障排查

```bash
# 看 plugin 是否被 OpenClaw 加载
openclaw plugins list | grep memoryd

# 看 capture 是否成功
tail -f ~/.local/share/memoryd/logs/openclaw-events.log

# 走 FS-watch 兜底（不依赖 plugin）
memoryd mirror --openclaw --once
memoryd list --source=openclaw --recent=5
memoryd list --source=openclaw-fs --recent=5
```
